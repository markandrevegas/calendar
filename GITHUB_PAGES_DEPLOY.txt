GitHub Pages setup for this project

Files added:
- .github/workflows/deploy-pages.yml
- nuxt.config.ts updated to respect NUXT_APP_BASE_URL

What the workflow does:
- Runs on pushes to main and on manual dispatch
- Installs dependencies with Bun
- Builds the app with `bun run generate`
- Publishes `.output/public` to GitHub Pages
- Lets `actions/deploy-pages` use its default Pages deployment environment

What you need to do in GitHub:
1. Push this repository to GitHub.
2. Make sure your default deployment branch is `main`.
3. Open the repository on GitHub.
4. Go to Settings > Pages.
5. Under Build and deployment, set Source to `GitHub Actions`.
6. Push to `main` or run the workflow manually from the Actions tab.

Base path behavior:
- If this repo is published as `username.github.io`, the app uses `/`.
- If this repo is published as a project site like `username.github.io/calendar`, the workflow sets the base URL to `/<repo-name>/`.

Local commands:
- Start dev server: `bun dev`
- Generate static output: `bun run generate`

Published output directory:
- `.output/public`
